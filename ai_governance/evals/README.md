# PolicyPulse Evals (Day Zero)

This folder stores evaluation artifacts for prompt and model changes.

## Purpose

Evaluation artifacts are intended to support:

- regression testing for prompt changes
- red-team cases for safety and hallucination risk
- future bias and fairness checks
- governance review before prompt promotion

## Day Zero Scope

At this stage, the evaluation artifacts are static examples only. They are not yet wired into CI.

## Future Direction

Planned improvements:

- automated evaluation runs in CI
- minimum score thresholds
- regression failure gates
- version-linked prompt and model reporting
